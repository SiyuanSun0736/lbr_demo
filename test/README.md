# LBR 测试程序

这是一个用于测试 LBR (Last Branch Record) 功能的示例程序。

## 程序说明

`test_lbr.c` 包含多种分支场景，专门设计用来测试 LBR 的分支记录能力：

### 测试场景

1. **冒泡排序** - 嵌套循环和条件分支
   - 演示内层循环的条件判断
   - 产生大量的分支预测和跳转

2. **数字分类** - 多重if-else分支
   - 演示多个连续的条件判断
   - 测试不同的执行路径

3. **斐波那契递归** - 函数调用分支
   - 演示递归调用产生的分支
   - 测试深度函数调用栈

4. **命令处理** - Switch多路分支
   - 演示switch语句的分支行为
   - 测试跳转表的实现

5. **偶数求和** - 循环+条件组合
   - 演示循环内的条件判断
   - 测试循环优化

6. **密集分支操作** - 随机分支测试
   - 大量的条件分支
   - 测试分支预测失败

## 编译和运行

### 编译测试程序

```bash
cd test
make
```

### 单独运行测试程序

```bash
make run
```

### 使用 LBR 监控测试程序

#### 方法1: 在一个终端运行监控程序

```bash
# 终端1: 启动LBR监控
cd /home/ssy/lbr
make run
```

监控程序会监听所有的 `execve` 系统调用，当新程序启动时会捕获其 LBR 信息。

```bash
# 终端2: 运行测试程序
cd /home/ssy/lbr/test
./test_lbr
```

#### 方法2: 一体化脚本

创建一个脚本来自动化整个流程：

```bash
cd /home/ssy/lbr
# 在后台启动LBR监控
sudo ./lbr-demo &
LBR_PID=$!

# 等待监控程序初始化
sleep 2

# 运行测试程序
cd test
./test_lbr

# 等待捕获完成
sleep 3

# 停止监控程序
sudo kill $LBR_PID
```

## 预期输出

### 测试程序输出

程序会输出各个测试场景的结果，例如：
```
=== LBR 测试程序开始 ===
PID: 12345

测试1: 冒泡排序
排序前: 64 34 25 12 22 11 90 88 45 50
排序后: 11 12 22 25 34 45 50 64 88 90

测试2: 数字分类
...
```

### LBR 监控输出

监控程序会输出捕获的分支记录，包括：
- 分支源地址 (FROM)
- 分支目标地址 (TO)
- 符号名称（如果可用）
- 可能的反汇编代码

示例输出：
```
LBR demo is running. Press Ctrl-C to exit.
Captured LBR data for PID: 12345
Entry 0:
  FROM: 0x401234 -> TO: 0x401567
  Symbol: bubble_sort+0x45
  ...
```

## 分析要点

使用这个测试程序可以观察：
1. 不同类型的分支指令（条件跳转、无条件跳转、调用、返回）
2. 分支预测的成功率
3. 热点代码路径
4. 函数调用关系
5. 循环的执行模式

## 注意事项

- LBR 监控需要 root 权限
- LBR 记录数量有限（通常16-32条），只记录最近的分支
- 某些编译器优化可能会改变分支行为
- 程序末尾的 `sleep(5)` 用于给监控程序足够的时间捕获数据
