在 pmu_monitor_all.c 之外，你还可以采集很多与性能/系统状态有关的数据，按用途分列出并简要说明如何实现：

- **调用栈/调用链（Callstack / Callchain）**：通过 perf 的采样模式（`sample_type` 加 `PERF_SAMPLE_CALLCHAIN`）或使用 eBPF（libbpf）采集用户/内核栈，便于将事件归到函数符号上。  
- **分支记录（LBR / Branch Stack）**：开启 `PERF_SAMPLE_BRANCH_STACK` 或使用 CPU 的 LBR 硬件功能，获取分支历史用于热点与控制流分析。  
- **采样 IP / 指令地址（IP/PC）**：在采样中记录 `PERF_SAMPLE_IP`，再做符号解析（addr2line / DWARF）得到函数/源代码位置。  
- **原始事件（RAW events）与详细 PMU 信息**：增加更多 raw event（你已有），并记录 `PERF_SAMPLE_RAW` 以获取硬件返回的原始数据。  
- **时间戳与高精度时间序列**：记录精确时间（已做），也可使用 TSC 或 `PERF_SAMPLE_TIME` 以统一时间基准并做跨主机/跨核对齐。  
- **进程/线程维度（PID/TID / cgroup）**：采样时记录 `PERF_SAMPLE_TID`、CPU 和 cgroup 信息，按线程或容器聚合统计。  
- **频率/周期信息（sample period / call frequency）**：改为基于样本周期触发（`sample_period`）而非固定 timerfd，或同时记录事件的周期/频率。  
- **上下文切换 / 调度事件 / 系统调用**：通过 tracepoints、perf trace 或 eBPF 跟踪 `sched_switch`、`sys_enter`/`sys_exit` 等，用于分析延迟与阻塞。  
- **内存/页面指标**：读取 `/proc/<pid>/statm|smaps`、page faults、TLB miss 细节或 NUMA 访问信息。  
- **能耗/功耗（RAPL / power domains）**：使用 RAPL 或 Intel PCM（或 perf 的 energy events）采集功耗/能量数据。  
- **温度/频率/CPU 信息**：读取 `/sys/devices/system/cpu/...`、CPU freq 和温度传感器，分析频率缩降与热节流影响。  
- **汇总/累计 vs 增量**：当前程序重置计数器以记录“每间隔增量”，可改为不 reset（记录累计），或在程序层做增量/滑动窗口/累加。  
- **事件上下文（stack frame、寄存器、raw samples）**：在采样时请求 `PERF_SAMPLE_REGS_USER`、`PERF_SAMPLE_STACK_USER` 以获得寄存器与栈帧快照（更重，影响开销）。  
- **自定义打点/用户事件（user markers）**：在被测程序插入 tracepoints/软件事件，或用 `perf_event_open` 的 `PERF_SAMPLE_REGS_INTR`/`mmap` 通道传递自定义元数据。  
- **网络/IO/存储指标**：结合系统统计（net/dev、iostat）或 tracepoints 收集网络包、磁盘延迟等外部瓶颈数据。

如果要把其中一项加到 pmu_monitor_all.c，常见低成本改动：
- 开启 `pe.sample_type |= PERF_SAMPLE_CALLCHAIN` 并设置 `sample_period`，改用 `mmap` 环形缓冲读取样本（可获得 callchain、ip、time、tid 等）。  
- 或集成一个简单 eBPF 程序（libbpf）采样 callstack/LBR，再把结果写回日志。
